# poe2tracker

Автономный трекер экономики и билдов Path of Exile 2. Собирает данные в SQLite, считает
выводы (моментум цены, объём Busy/Steady/Quiet, всплески, «горячие» позиции) и шлёт
отфильтрованные уведомления в Discord/Telegram. Крутится сам на **серверах GitHub** по
расписанию — твой ПК держать включённым не нужно.

## Что где
- `tracker.py` — экономика: poe2scout (цены + **объём**), анализ, алерты, авто-сверка путей API,
  экспорт `docs/data.json` для дашборда.
- `builds.py` — билды: poe.ninja, выборка top-N **по популярности** (30% → top200, 0.1% → top3),
  агрегация баз/модов/слотов по каждому асцендансу.
- `craft.py` — **ожидаемая стоимость крафта** через Монте-Карло в текущих ценах БД (среднее/медиана/p90).
- `run.py` — единая точка входа (её дёргает cron).
- `docs/index.html` — веб-дашборд (GitHub Pages). Читает `docs/data.json`, есть демо-fallback.
- `.github/workflows/track.yml` — расписание (каждые 30 мин; билды сами гейтятся на 12ч).

## Проверить логику без сети
```bash
python run.py --selftest      # экономика + билды + крафт на синтетике
python run.py --craft         # демо-оценка стоимости крафта в ценах БД
```

## Веб-дашборд (GitHub Pages)
Каждый прогон пишет `docs/data.json` и коммитит его обратно. Чтобы он был виден в вебе:
Settings → Pages → Source: `Deploy from a branch`, ветка `main`, папка `/docs`.
Дашборд откроется на `https://<user>.github.io/<repo>/`. Пока данных нет — покажет демо.

## Запуск «в облаке» (не у себя) — GitHub Actions
1. Создай приватный репозиторий и залей туда эти файлы.
2. Settings → Secrets and variables → Actions → **New repository secret**, добавь:
   - `POE2_CONTACT` — твой email (poe2scout требует контакт в User-Agent).
   - `POE2_LEAGUE` — точное имя текущей лиги (сверь на poe2scout.com).
   - `POE2_DISCORD_WEBHOOK` — вебхук канала Discord (или пара Telegram ниже).
   - `POE2_TELEGRAM_TOKEN` + `POE2_TELEGRAM_CHAT` — если хочешь Telegram вместо Discord.
3. Вкладка **Actions** → включи workflow → кнопка **Run workflow** для первого прогона.
   Дальше он идёт по расписанию сам. Алерты падают в твой канал.

Порог алертов, окно, категории и параметры масштабирования top-N правятся вверху
`tracker.py` (ALERT_RULES, WATCH) и `builds.py` (SCALE, BUILDS_REFRESH_HOURS).

## Локальный запуск (если всё же захочешь у себя)
```bash
pip install -r requirements.txt
export POE2_CONTACT="you@example.com"
export POE2_LEAGUE="<текущая лига>"
python run.py --once           # один цикл
python run.py --report         # выводы из накопленной БД
```

## Важные оговорки (честно)
- **Пути API нужно сверить.** poe2scout и особенно poe.ninja PoE2 могут отличаться регистром/
  структурой. Парсеры терпимы к вариациям полей, но эндпоинты вынесены в константы
  (`SCOUT_ENDPOINTS`, `NINJA_EP`) — при первом запуске проверь ответы (`tracker.py --discover`)
  и при необходимости поправь одну строку.
- **Легальность.** Мы читаем агрегаторы (poe2scout/poe.ninja), а не долбим официальное trade API
  GGG ради БД — это против их rate-limit/ToS. Никакой автоматизации действий в игре: только
  сбор данных и уведомления. Автопокупка/автовиспер — это читерский софт и путь к бану, здесь его нет.
- **Вежливость к poe.ninja.** Билды обновляются раз в 12ч (не каждые 30 мин) и с общим потолком
  запросов персонажей за прогон — иначе выборка top200 по всем классам превратится в абьюз.
- **Спрос до баз/модов/слотов** считается по фактической экипировке топ-чаров выборки —
  это оценка «что реально носят», а не гарантия. Чем нишевее класс, тем меньше выборка и шумнее %.
- **Крафт-оценка** зависит от `p_hit` — вероятности, что слэм даст нужный мод. Это НЕ угадывается:
  бери веса модов с poe2db (вес нужного мода / сумму весов пула на слот). Симуляция честно
  показывает не только среднее, но и p90 — хвост дорогих неудач, который и есть реальный риск.
- Данные poe2scout покрывают только высоколиквидные предметы (валюта/руны/эссенции/уники);
  конкретный gear там пустой — это by design, спрос на gear закрывает модуль билдов.
